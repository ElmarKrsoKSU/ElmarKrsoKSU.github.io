About the Author

Hi, my name is Elmar Krso and I am a cybersecurity student at Kennesaw State University. I picked web servers as the topic for this project because they are something I use every day without really thinking about, and I wanted to learn more about how they actually work behind the scenes. There is also a lot of overlap with the security stuff I'm studying, especially around HTTPS, encryption, and how reverse proxies handle DDoS attacks.

About This Project

This site was built for IT3203 (Web Application Development) at KSU. The goal was to research a web-related topic and build a multi-page website that demonstrates responsive design, mobile-friendly layout, and interactive features. The whole thing is hand-coded in HTML, CSS, and JavaScript with no frameworks.

The site covers three main areas: the history of web servers from 1990 to today, the modern landscape with cloud computing and CDNs, and the key technical concepts that tie it all together. There is also an interactive quiz with automatic grading and a progress bar.

What I Learned

Some of the bigger things I picked up while building this:

  • Responsive design is harder than it looks. Getting the hamburger menu, floated images, and tables to all behave correctly across desktop, tablet, and mobile took more iterations than I expected.
  • Separation of concerns matters. Moving all the styles into one CSS file and all the scripts into one JS file made everything easier to maintain.
  • Web servers have a really interesting history. I had no idea Apache was named after "a patchy server" or that NGINX was created in Russia to solve scaling problems for a search engine.
  • Small details add up. Things like making sure tap targets are big enough, escaping HTML entities in JavaScript output, and making the menu close when you tap outside it - none of it is hard individually, but it all matters for a good experience.
